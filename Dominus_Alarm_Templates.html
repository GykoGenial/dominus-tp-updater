4c
HARSI SL — Universal VOLLAUTOMATIK v4.11
Ein einziger Watchlist-Alarm ersetzt Alarm 4/4b für alle Symbole — Railway ermittelt die Richtung aus der offenen Position.
🎯 Was dieser Alarm löst
Statt Alarm 4 (Long) + Alarm 4b (Short) pro Symbol manuell einzurichten, genügt 1 einziger Watchlist-Alarm vom Typ Any alert() function call. Die Pine feuert bei jedem H2-Close mit HARSI-Kreuzung und embedded den Preis direkt im JSON-Body — das bekannte {{plot(...)}}-Problem bei Watchlist-Alarmen ist umgangen. Railway filtert stumm alle Symbole ohne offene Position.
⚠ Voraussetzung: Pine-Snippet einfügen
Im DOMINUS Orchestrator Pine am Ende (nach den bestehenden alertcondition-Zeilen) folgenden Code einfügen: 
// v4.11 HARSI_SL Universal-Emitter — feuert direkt via alert()
// HARSI ist rsi_overlay (RSI Overlay plot). Ausstiegslinien ±20.
if barstate.isconfirmed
    if ta.crossover(rsi_overlay, -20)   // Long-Stop: HARSI verlässt Oversold aufwärts
        alert('{"symbol":"' + syminfo.ticker + '","side":"long","signal":"HARSI_SL","price":' + str.tostring(close, "#.########") + ',"timeframe":"' + timeframe.period + '"}', alert.freq_once_per_bar_close)
    if ta.crossunder(rsi_overlay, 20)   // Short-Stop: HARSI verlässt Overbought abwärts
        alert('{"symbol":"' + syminfo.ticker + '","side":"short","signal":"HARSI_SL","price":' + str.tostring(close, "#.########") + ',"timeframe":"' + timeframe.period + '"}', alert.freq_once_per_bar_close)
Alternative: Falls du side nicht mitsenden willst (z.B. wenn Pine die Richtung nicht sauber trennen kann), setze stattdessen "side":"auto". Railway ermittelt die Richtung dann selbst aus der offenen Position.

Sling-SL — Universal NEU v4.12
Swing-Pivot (3/3) auf H2 — SL wird dynamisch an den letzten bestätigten Pivot nachgezogen, nur protektiv.
🎯 Was dieser Alarm löst
Die DOM-ORC v2.3 erkennt Sling-Pivots (Swing-High/Low mit 3 Bars links & 3 Bars rechts) und feuert bei jedem bestätigten Pivot einen SLING_SL-Alarm. Sling-High → SL für Shorts, Sling-Low → SL für Longs. Railway zieht den SL nur protektiv nach (nie in Richtung Verlust) und verwendet einen ATR-Fallback: max(0.8 %, 0.5 · ATR(14)) — zu enge Stops werden automatisch auf diesen Puffer aufgeweitet.
⚠ Voraussetzung: DOM-ORC v2.3 im Chart
Der Sling-SL-Emitter ist bereits fester Bestandteil der Dominus_Orchestrator_v2.3.pine. Kein zusätzliches Snippet nötig — einfach die neue Pine-Version laden und den bestehenden DOM-ORC im Chart ersetzen.

ph = ta.pivothigh(high, sling_left, sling_right)   // Sling-High (Short-SL)
pl = ta.pivotlow (low,  sling_left, sling_right)   // Sling-Low  (Long-SL)
atr_val = ta.atr(sling_atr_len)

if barstate.isconfirmed and not na(ph)
    alert('{"symbol":"' + syminfo.ticker + '","side":"short","signal":"SLING_SL","pivot":' + str.tostring(ph) + ',"price":' + str.tostring(close) + ',"atr":' + str.tostring(atr_val) + ',"timeframe":"' + timeframe.period + '"}', alert.freq_once_per_bar_close)

if barstate.isconfirmed and not na(pl)
    alert('{"symbol":"' + syminfo.ticker + '","side":"long","signal":"SLING_SL","pivot":' + str.tostring(pl) + ',"price":' + str.tostring(close) + ',"atr":' + str.tostring(atr_val) + ',"timeframe":"' + timeframe.period + '"}', alert.freq_once_per_bar_close)
Parameter (im DOM-ORC einstellbar): sling_left = 3 · sling_right = 3 · sling_atr_len = 14. Visualisierung: rote gestrichelte Linie = Sling-High, grüne = Sling-Low.

2. ATR-Fallback: max(0.8 %, 0.5 · ATR(14)) — schmale Stops werden automatisch aufgeweitet

6. DCA Auto-Void: Wenn SL einen DCA-Level überschreibt (SL im Gewinn = logisch unmöglich), werden die betroffenen DCAs automatisch gelöscht

8. Max Hebel: 25x — harte Obergrenze, Formel Hebel = 25 / SL-Abstand% wird gekappt
9. Max Exposure: 25 % Equity (inkl. Hebel) — per_order · 3 · Hebel ≤ balance · 0.25 · 10, Warnung in /trade

Ergänzung zu HARSI_SL — beide Alarme koexistieren; immer der protektivere SL gewinnt.
Schutz vor Stop-Hunts — ATR-Fallback verhindert, dass zu enge Pivots den SL direkt an den Markt kleben.

Was bedeuten die drei Parameter?
⚡
Hebel (Leverage) Wikipedia ↗
Der Hebel multipliziert sowohl Gewinne als auch Verluste. Bei 10x Hebel bedeutet eine Kursbewegung von 1% einen Gewinn oder Verlust von 10% auf die eingesetzte Margin.

DOMINUS-Formel: Empfohlener Hebel = 25 / SL-Abstand%
Beispiel: SL 2.5% vom Entry entfernt → Hebel = 25 / 2.5 = 10x

Die 25 steht für den maximalen Verlust von 25% der Margin bei SL-Auslösung — unabhängig vom gewählten Hebel. Ein zu hoher Hebel bedeutet: schon kleine Kursbewegungen lösen den SL aus.
Warnung ausgelöst wenn: gesetzter Hebel weicht um mehr als 2× vom Optimum ab — z. B. 20x statt empfohlener 12x.
📊
Kelly-Kriterium (Positionsgrösse) Wikipedia ↗
Das Kelly-Kriterium berechnet die mathematisch optimale Positionsgrösse um das langfristige Kapitalwachstum zu maximieren, ohne das Risiko eines Totalverlustes einzugehen.

Formel: f* = (p × b − q) / b
p = Winrate (z. B. 0.55), q = 1−p, b = Gewinn/Verlust-Verhältnis

In der Praxis wird oft der Half-Kelly (halbe Grösse) verwendet, um Schwankungen zu dämpfen. Railway zeigt den Kelly-% und den Half-Kelly-% im Trade-Setup-Bericht.
Warnung ausgelöst wenn: Initial-Margin der ersten Order übersteigt den Half-Kelly-Betrag. Mit DCA (1.5× + 2.5× Nachkauf) würde das Gesamtexposure dann ein Vielfaches des Optimums erreichen.
📉
Chance-Risiko-Verhältnis (R:R Ratio) Wikipedia ↗
Das R:R-Verhältnis setzt den möglichen Gewinn (bis TP) in Relation zum möglichen Verlust (bis SL). Ein R:R von 2.0 bedeutet: für jeden riskierten Franken wird 2 Franken Gewinn angestrebt.

Beispiel: Entry 100, SL 98 (−2%), TP 104 (+4%) → R:R = 4/2 = 2.0

Ein R:R von mindestens 1.5 ist Minimum (Railway-Variable MIN_RR). Darunter lohnt sich der Trade statistisch nicht, selbst bei guter Winrate — die schlechten Trades fressen die guten auf.
Warnung ausgelöst wenn: R:R < MIN_RR (Standard: 1.5). Abhilfe: SL enger setzen oder einen besseren Entry abwarten. Das Setup wird trotzdem aufgebaut — die Entscheidung liegt bei dir.
Keine Warnung = alles im grünen Bereich. Railway sendet die Abweichungs-Meldung nur wenn mindestens ein Parameter ausserhalb des Optimums liegt. Bei einem perfekten Setup erscheint nur der normale Trade-Bericht.
